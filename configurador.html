Instrucciones importantes

Esta URL debe estar configurada en Google Cloud Console como "Fuente autorizada". Si no ves tu URL aquí, asegúrate de estar accediendo desde GitHub Pages.

Configuración de OAuth2

Paso 1: Verificar en Google Cloud

  1. Ve a Google Cloud Console - Credenciales
  2. Haz clic en tu Client ID OAuth 2.0
  3. Verifica que en "Fuentes de JavaScript autorizadas" aparezca tu URL
  4. Si no aparece, haz clic en "Editar" y agrégala
Instrucciones Detalladas para Resolver "no registered origin"

Error: "no registered origin"

Causa: Google no reconoce la URL desde donde intentas acceder.

Solución:

  1. Abre Google Cloud Console
  2. Ve a "APIs y servicios" → "Credenciales"
  3. Haz clic en tu Client ID OAuth 2.0
  4. En "Fuentes de JavaScript autorizadas", agrega esta URL:
URL a agregar:
https://tu-usuario.github.io

Reemplaza "tu-usuario" por tu nombre real de usuario de GitHub

Error: "Error 401: invalid_client"

Causa: El Client ID configurado no es correcto.

Solución:

  1. En Google Cloud Console, ve a Credenciales
  2. Haz clic en tu Client ID para ver los detalles
  3. Copia el "ID de cliente" completo
  4. Pégalo en el campo "Client ID" arriba

Formato correcto: 123456789-abcdefghijklmnop.apps.googleusercontent.com

Verificación Final

Después de configurar:

  1. Haz clic en "Probar Configuración"
  2. Si ves "✅ Configuración válida", haz clic en "Iniciar Aplicación"
  3. Se abrirá el sistema completo de asistencia
